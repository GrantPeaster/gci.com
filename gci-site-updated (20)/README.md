# Georgia Civil — Website

A fully static site (HTML, CSS, JS, images). No build step, no server code.
It can be hosted for free on **GitHub Pages**.

---

## Deploy to GitHub Pages

1. **Create a repo** on GitHub and upload everything in this folder to it
   (keep the structure — `index.html` must be at the repo root, alongside
   `assets/`, `favicon.ico`, `.nojekyll`, etc.).
2. In the repo, go to **Settings -> Pages**.
3. Under **Build and deployment -> Source**, choose **Deploy from a branch**.
4. Set the branch to **main** and the folder to **/ (root)**, then **Save**.
5. Wait ~1 minute. Your site will be live at
   `https://<your-username>.github.io/<repo-name>/`.

All links and asset paths are **relative**, so the site works whether it's served
from the domain root or from a `/<repo-name>/` subfolder — no changes needed.

The included **`.nojekyll`** file tells GitHub Pages to serve the files exactly as
they are (GitHub otherwise runs them through Jekyll, which can hide folders).

### Custom domain (optional)
In **Settings -> Pages -> Custom domain**, enter your domain (e.g. georgiacivil.com)
and follow GitHub's DNS instructions. GitHub Pages provides free HTTPS.

## Contact / careers forms (IMPORTANT)

The two forms (Contact page + About page) previously used Netlify Forms, which only
work on Netlify. They've been switched to **Web3Forms** — free, unlimited
submissions, no backend, works on GitHub Pages.

To turn the forms on:
1. Go to https://web3forms.com, enter the email address where you want form
   submissions delivered, and copy the free **Access Key**.
2. Open `contact.html` and `about.html`, find this line in each form:
       <input type="hidden" name="access_key" value="YOUR_WEB3FORMS_ACCESS_KEY">
   and replace YOUR_WEB3FORMS_ACCESS_KEY with your key.
3. Commit/push. Submissions will arrive in your inbox. The form shows a
   "thanks" message on success without leaving the page.

Until the key is set, the forms validate but won't deliver.

Alternatives if you prefer: Formspree (free tier) — swap the form `action` to your
Formspree endpoint. Or a no-service fallback: replace the form with a `mailto:` link.

## Updating content

- **Documents:** drop the registration PDFs into `assets/docs/` using the filenames
  listed in `assets/docs/README.txt`; the download links then work automatically.
- **Project / leadership photos:** add images to `assets/img/` and update the
  matching background-image / src references.
- **Navigation & footer** are generated by `assets/js/main.js` (the NAV array and
  buildFooter).

## Note on the QR codes

The printed-card QR codes point to gcipitch.netlify.app. After you move to a new
address (a github.io URL or a custom domain), **regenerate the QR codes** against
the new URL before printing anything. For permanence, consider pointing them at a
short redirect you control so the destination can change without reprinting.
